# Week 1 Learning Log

## What did I learn in class this week?

This week focused on understanding the structure and philosophy of the course rather than jumping straight into heavy coding.

We reviewed the syllabus and grading breakdown:

- Checkpoints (40%) – mini projects, exercises, learning logs  
- Understanding Assessment (20%) – paper quizzes + in-person interviews  
- Final Project (25%) – main portfolio piece  
- Participation (15%) – in-class engagement, office hours, collaboration  

The biggest conceptual shift introduced was:

**Old way of learning programming:**  
- Memorize syntax  
- Avoid documentation  
- Code independently  

**New way (this course's mindset):**  
- Focus on understanding core concepts  
- Use documentation effectively  
- Use AI as a productivity and thinking tool  

This reframed programming as problem-solving and system thinking rather than just memorizing commands.

We also:

- Set up GitHub (created repository: `oim3640`)
- Practiced committing and pushing changes
- Learned basic terminal commands:
  - `cd` (navigate folders)
  - `cd ..` (move up directory)
  - `ls` / `dir` (list files)
  - `mkdir` (create folder)
  - `pwd` (print working directory)

Additionally, we discussed moving from **problem → prototype**, shared AI-generated app ideas, and evaluated whether solutions actually solve real problems.

Overall, I learned that this course is less about writing random code and more about structured thinking, iteration, and building real, functional systems.

---

## What challenges did I bump into?

1. **Shifting mindset**
   The hardest part wasn’t technical — it was mental. Letting go of the idea that I need to memorize everything and instead focusing on understanding concepts + using AI strategically feels different.

2. **GitHub workflow**
   Understanding the full flow (edit → commit → push) required slowing down and thinking about version control as a system instead of just “saving.”

3. **Terminal navigation**
   The command line is simple, but it requires precision. One wrong directory and nothing works. I realized I need more repetition to feel fully confident navigating without hesitation.

4. **Evaluating AI output**
   When generating an app idea with AI, it was challenging to think critically about:
   - Does this actually solve a problem?
   - What features are missing?
   - Is this scalable?

This week made it clear that technical skills and critical thinking go hand-in-hand.

---

## Reflection

Week 1 set the tone: this is not a “copy code and pass” class. It’s about building systems, thinking clearly, and using tools intelligently. The foundation feels strong, and I understand what’s expected moving forward.
